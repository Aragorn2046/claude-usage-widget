◉ Claude Usage Monitor

Show your Claude.ai usage percentage on Windows or Stream Deck. Pure PowerShell, no dependencies.

1 What You Need

  • Windows 10 or 11
  • Claude Pro or Max subscription on claude.ai
  • Claude Code (free CLI tool) — this generates the credentials the scripts use
  • Stream Deck + "Text File Tools" plugin (only for the Stream Deck version)

2 Get Your Credentials (One-Time Setup)

The scripts read credentials that Claude Code creates automatically when you log in. You do NOT need any API keys or tokens — just install Claude Code and authenticate.

Step A — Install Claude Code

  1. Open a terminal (PowerShell or Command Prompt)
  2. Run: 
    npm install -g @anthropic-ai/claude-code

    (Requires Node.js 18+. If you don't have npm, install Node.js first.)

Step B — Authenticate

  1. Run: 
    claude
  2. A browser window will open. Sign in with your Claude.ai account (the same Pro/Max account).
  3. Approve the permissions when prompted.
  4. Done. You can close Claude Code now.

This creates a credentials file at:

C:\Users\YOUR_USERNAME\.claude\.credentials.json
⚠ Do NOT share this file. It contains your personal OAuth tokens. The scripts read it automatically — you never need to open or edit it.

Where Are The Keys?

There are no manual keys to copy or paste. The credential file is created and managed automatically by Claude Code. It contains:

  • accessToken — short-lived token for API calls (auto-refreshed by the scripts)
  • refreshToken — long-lived token used to get new access tokens
  • expiresAt — when the access token expires
  • subscriptionType — your plan (pro, max, etc.)

If the scripts ever show ? or an error, just run claude in a terminal again to refresh your authentication.

3 Choose Your Version

💻 Windows Popup windows/
Click to see a dark popup with both 5-hour and 7-day usage bars. Auto-closes after 10 seconds.
🎮 Stream Deck Live streamdeck/
Always-on display. Shows usage % directly on Stream Deck buttons, updating every 60 seconds.

4 Setup: Windows Popup

Copy the windows/ folder anywhere you like.

Run manually

Double-click usage.ps1, or run:

powershell.exe -ExecutionPolicy Bypass -File "C:\path\to\windows\usage.ps1"

Run from Stream Deck

Add a System > Open action with this command:

powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\path\to\windows\usage.ps1"

Controls

  • ESC or click anywhere to close
  • Auto-closes after 10 seconds

5 Setup: Stream Deck Live Display

Step A — Install the plugin

  1. Open the Stream Deck app
  2. Go to the Stream Deck Store (bottom-right corner)
  3. Search for "Text File Tools" by BarRaider
  4. Install it (free)

Step B — Copy the files

Copy the streamdeck/ folder to a permanent location, for example:

C:\Users\YOUR_USERNAME\StreamDeck\claude-usage\

Step C — Start the background monitor

Add a Stream Deck button (System > Open) with this command:

powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\path\to\streamdeck\usage-monitor.ps1"

Press it once to start the monitor. It runs silently in the background and prevents duplicate instances, so pressing it again is harmless.

Step D — Add the display buttons

Drag "Text File Monitor" (from Text File Tools) onto two buttons:

5-Hour Button
File path:
C:\...\streamdeck\usage-5h.txt
7-Day Button
File path:
C:\...\streamdeck\usage-7d.txt

Set the button title to "5h" and "7d" respectively so you can tell them apart. The percentage will update every 60 seconds.

Optional: Auto-start at login

  1. Press Win + R, type shell:startup, press Enter
  2. Create a shortcut in that folder pointing to: 
    powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\path\to\streamdeck\usage-monitor.ps1"
  3. The monitor will now start automatically when you log in.

6 Troubleshooting

Button shows ? Token expired or credentials missing. Run claude in a terminal to re-authenticate.
Button shows ERR Network error or API issue. Check your internet connection. The monitor will retry every 60 seconds.
Button not updating The monitor process may have stopped. Press the start button again. If it still doesn't work, open Task Manager, end any powershell.exe running usage-monitor.ps1, then re-launch.
Popup shows error Same fix: run claude in a terminal to refresh authentication.
